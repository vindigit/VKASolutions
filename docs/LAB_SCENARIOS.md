# Lab Scenarios

## Scenario 1: CEO Account Locked

- User: Vindon Alexander
- Goal: Verify identity, unlock access, document high-priority handling
- Technician practice: escalation discipline, executive support etiquette, account recovery

## Scenario 2: Sales User Missing CRM Access

- User: Sonia Patel
- Goal: Diagnose why workstation access works but CRM access fails
- Technician practice: role mapping, entitlement checks, validation after remediation

## Scenario 3: Duplicate Customer Record

- Customer: Apex Logistics / Apex Logistics LLC
- Goal: Identify duplicate records and walk through the merge playbook
- Technician practice: data hygiene, evidence gathering, business impact awareness

## Scenario 4: Export Button Error

- Customer: Northwind Fiber
- Goal: Reproduce and document application failure details
- Technician practice: issue replication, bug intake quality, user communication

## Scenario 5: Password Reset Request

- User: standard business user
- Goal: complete a routine password reset workflow
- Technician practice: identity verification, reset steps, documentation

## Scenario 6: New Employee Onboarding

- User: Maya Chen
- Goal: ensure a new hire has CRM access and correct default role assignment
- Technician practice: provisioning checklist execution, first-day readiness

## Scenario 7: Terminated Employee Access Removal

- User: Trevor Mills
- Goal: remove remaining CRM access during offboarding
- Technician practice: access revocation, security awareness, checklist completion
